Release Notes for the Examples

System Requirements

 

  • Microsoft Windows 8, Windows Server 2012, Windows 7 with Service Pack 1, or Windows Server 2008 R2 with Service Pack 1 (32 bit or 64 bit editions).
  • Microsoft .NET Framework version 4.0.
  • Microsoft Visual Studio 2010 Ultimate, Premium, or Professional edition with Service Pack 1 installed, or Visual Studio 2012 Ultimate, Premium, or Professional edition.
  • Microsoft SQL Server 2012, SQL Server Express 2012, SQL Server 2008, or SQL Server Express 2008.
    - see the notes below for configuration information and compatibility with your operating system.
  • Windows Azure SDK version 1.8 (October2012) for .NET (includes the Visual Studio Tools).
    - see the notes below for more information, or if you do not already have this installed.
  • ASP.NET MVC 4 Framework.
  • Windows Identity Foundation. This is required for claims-based authorization.
  • WebAii testing framework. This is required only if you want to run the functional tests. Place the assembly ArtOfTest.WebAii.dll in the Lib\WebAii folder of the examples.
  • Additional components and frameworks required by the examples are installed using NuGet when you run the solutions. Ensure you have configured NuGet to download missing packages.

This list includes third-party software not provided by Microsoft. The links are designed to help you to obtain the required software from the appropriate vendor's website. The licensing terms and conditions for such software will differ from that of the Microsoft software in the list. We acknowledge the help provided by third-party vendors where their software is used in this project.

 

 

Check for and Install Required Files and Dependencies

 

Before you start, ensure you have a compatible version of SQL Server installed on your computer:

  • For Windows 8 and Windows Server 2012: SQL Server 2012 or SQL Server Express 2012
  • For Windows 7 and Windows Server 2008: SQL Server 2012, SQL Server Express 2012, SQL Server 2008, or SQL Server Express 2008

The Dependency Checker utility requires the instance name of the SQL Server. The default is SQLExpress, and so the Dependency Checker will install the sample databases into the local SQL Server Express instance. If you are using SQL Server instead of SQL Server Express, or you want to install the database for the samples in a named (non-default) instance, you must edit the configuration file used by the Dependency Checker utility. Open the file DependencyChecker.exe.config from the Setup\DependencyChecker folder and edit the value for sqlInstanceName shown highlighted here:

<dependency check="DbSetup"
  enabled="true"
  category="Required"
  title="Database for samples"
  explanation="...."
  scriptName="..."
  settings="sqlInstanceName=SQLEXPRESS!alias=WAGSqlAlias!db=..."
/>

After you have confirmed that SQL Server or SQL Server Express is installed and that the configuration file contains the correct server name, run the CheckDependencies.cmd command file that is located in the root folder of the examples to check for software prerequisites required. You must run this using an account that has Administrator privileges. The Dependency Checker reports any components missing on your system, and provides links to obtain, install, and configure these missing components. You must have the .NET Framework version 4.0 installed to run the Dependency Checker.

 

NOTE: If you want to install the Windows Azure SDK for .NET on Windows Server 2008 R2, you must first ensure that the .NET Framework version 3.5.1 is installed on your computer.

 

 

Installing the Correct Version of the Windows Azure SDK

 

The examples require version 1.8 (October 2012) of the Windows Azure SDK. You can install this alongside later versions. The Dependency Checker will check if version 1.8 is installed. If not, it will provide a link to install the SDK. After installation is complete, open the About Microsoft Visual Studio dialog from the Help menu and verify that the list of installed products includes Windows Azure Tools October 2012.

 

In some circumstances the Dependency Checker may fail to detect that version 1.8 of the Windows Azure SDK is installed, even after prompting to install it and the installation is successful. If you are sure that the version 1.8 SDK is installed you can ignore the warning displayed in the Dependency Checker.

 

If the Dependency Checker fails to install the version 1.8 SDK you must install it manually. Note that you cannot install this using the Web Platform Installer link on the download page because it always defaults to installing the latest version. Instead, use the DOWNLOAD links on the page Windows Azure SDK for .NET - October 2012 to install either the x86 or x64 versions of the Authoring Tools, Windows Azure Libraries for .NET, and Windows Azure Tools for Visual Studio (either VS 2010/VS100 or VS 2012/VS110). If you already have a later version installed, you do not need to install the version 1.8 Windows Azure Emulator because the solutions will run using the later version of the emulator.

 

NOTE: If you have a later version of the SDK installed, the Visual Studio properties page for a cloud project will provide a link to upgrade the project to the later version of the SDK. Do not upgrade the project. It must run under version 1.8 to avoid inconsistencies between versions.

 

 

Allow NuGet to Download Missing Packages

 

You must ensure that Visual Studio is configured to use the latest version of the NuGet package manager and allow it to download missing packages during the build process. To enable this:

  1. Open the Visual Studio Extension Manager dialog from the Tools menu.
  2. Check that the NuGet Package Manager is installed. If it is not already installed select Online Gallery and install it. You can use the Search feature to find it in the Gallery.
  3. If it is already installed select Updates and check if there is an update available. If there is, select Update to install it. If the update fails, uninstall the existing version and try again.
  4. Open the Visual Studio Options dialog from the Tools menu.
  5. Expand the section for Package Manager and select General.
  6. Ensure that Allow NuGet to download missing packages during build and Automatically check for updates are checked.

 

Known Issues When Running the Examples

 

You must run the projects and solutions under an account that has Administrator privileges.

 

When you run the example for the first time in the Compute Emulator you may encounter an error because your web browser opens automatically and makes a request to the application. This can occur while the start-up tasks that initialize Windows Azure local storage are still running. Simply reload the page in your browser to resolve this.
